# OpenHTTPA A2A (Agent-to-Agent)

The `openhttpa-a2a` crate provides the high-level semantic orchestration layer for the Zero-Trust Messenger ecosystem. It governs how autonomous AI agents discover, authenticate, and synchronize data with one another across untrusted network boundaries.

## Architecture

This crate integrates closely with two core sub-systems:

1. **AIQL (Agent Intelligence Query Language)**: The semantic firewall. Before any payload is processed or written to disk, it must pass through an AST-based declarative policy engine that verifies the `intent`, `caller_did`, and `caller_mrenclave`.
2. **SDMF (Secure Distributed Memory Fabric)**: The persistent storage backend. A2A uses `openhttpa-fabric` to ensure that any state generated by the agent is cryptographically sealed to the CPU's silicon measurement using hardware enclaves (Intel SGX/TDX) before hitting the disk.

## Quick Start

You can find a comprehensive runnable example in `examples/swarm_agent.rs`.

```bash
cargo run --example swarm_agent -p openhttpa-a2a
```

### Basic Workflow

```rust
use openhttpa_a2a::A2AAgent;
use openhttpa_aiql::{AiqlConfig, AiqlEngine, AiqlPolicy, Condition, PolicyAction};
use openhttpa_fabric::store::{KeyDerivationPolicy, MemoryStore, Topology};
use openhttpa_tee::mock::MockTeeProvider;
use std::sync::Arc;

// 1. Boot up the local Agent
let agent = A2AAgent::new("my_agent_id").unwrap();

// 2. Configure the Semantic Firewall
let mut aiql = AiqlEngine::new(AiqlConfig {
    default_action: PolicyAction::Quarantine,
    strict_mode: true,
});
aiql.load_policy(AiqlPolicy {
    name: "only_allow_trusted_sync".to_string(),
    condition: Condition::Equals {
        field: "intent".to_string(),
        value: "sync".to_string(),
    },
    action: PolicyAction::Allow,
    on_violation: None,
});

// 3. Mount the Secure Memory Fabric
let tee = Arc::new(MockTeeProvider::default());
let store = MemoryStore::new_rocksdb(
    Topology::Global,
    tee,
    "/tmp/rocksdb",
    KeyDerivationPolicy::PerTransaction
).unwrap();
```

## Security

By enforcing policies at the AST level _inside_ the hardware enclave memory space, `openhttpa-a2a` protects against malicious actors attempting to inject unauthorized intents or bypass the distributed ledger replication rules.
