This report details the temporal verification of the `OpenHTTPA` protocol using the **Tamarin Prover**. Unlike ProVerif, Tamarin specializes in verifying properties across stateful interactions and complex key-reveal scenarios.

## 1. Verified Lemmas

### ✅ Executability

- **Status**: Verified.
- **Description**: Ensures that there exists at least one valid trace where the protocol runs from start to finish. This proves the model is free of deadlocks or logical contradictions.

### ✅ Session Key Secrecy

- **Status**: Verified.
- **Description**: The session key derived during the handshake remains unknown to the adversary throughout the lifetime of the session.

### ✅ Forward Secrecy (Perfect Forward Secrecy)

- **Status**: Verified.
- **Description**: Even if the TEE's long-term signing key is compromised _after_ a session has ended, the historical session keys remain secure. This is achieved through the ephemeral ECDHE/ML-KEM key exchange.

### ✅ Injective Agreement (Replay Protection)

- **Status**: Verified.
- **Description**: Prevents replay attacks by ensuring that every client session corresponds to exactly one server session, uniquely identified by fresh nonces and the transcript hash.
